# Описание логики работы ИИ-агента
## Чат-бот-навигатор по базе знаний для водителей Яндекс Про

---

## 1. Назначение

ИИ-бот в Telegram, который помогает водителям Яндекс Про в 12 странах
быстро находить ответы на вопросы о выплатах, документах, правилах работы
и технических проблемах. Бот понимает произвольные формулировки и отвечает
на основе **380 реальных статей** базы знаний Яндекс Про.

---

## 3. Данные на входе и выходе

### Вход

| Источник            | Данные                        | Формат                                               |
|---------------------|-------------------------------|------------------------------------------------------|
| Водитель (Telegram) | Текст или выбор категории/FAQ | Сообщение или callback                               |
| Водитель (Telegram) | Выбор региона при /start      | Кнопки: 12 стран                                     |
| База знаний         | 380 статей с pro.yandex       | JSON: id, title, category, region, url_path, content |

### Выход

| Получатель | Данные                            | Формат                                                |
|------------|-----------------------------------|-------------------------------------------------------|
| Водитель   | Ответ + ссылка на реальную статью | Markdown + inline-кнопки 👍👎 В меню                  |
| SQLite     | Лог запроса                       | user_id, question, category, region, answer, feedback |

## 4. Инструменты и ИИ-сервисы

| Компонент               | Технология                               | Зачем                      |
|-------------------------|------------------------------------------|----------------------------|
| **Язык**                | Python 3.12 (Docker)                     | Стандарт                   |
| **Telegram-бот**        | aiogram 3.x                              | Асинхронный                |
| **LLM-генерация**       | YandexGPT Lite                           | Ответы, генерация вопросов |
| **Эмбеддинги**          | Yandex text-search-doc/query             | 256-мерные векторы         |
| **Векторный поиск**     | numpy cosine similarity + текстовый буст | Гибридный поиск            |
| **Векторное хранилище** | .npz + JSON                              | Без внешних БД             |
| **Классификация**       | Ключевые слова (12 регионов × 2 уровня)  | Бесплатно                  |
| **Rate limiter**        | Token bucket (9 req/s)                   | Ноль 429 ошибок            |
| **Логирование**         | SQLite                                   | Аналитика                  |
| **HTTP**                | httpx                                    | Асинхронные API-вызовы     |
| **Деплой**              | Docker + docker-compose                  | Переносимость              |

## 5. Индексация базы знаний

```
data/raw_articles/ (549 JSON-файлов)
        │
        ▼
  convert_kb.py: дедупликация, locale→region, region_urls
        │
        ▼
  data/knowledge_base.json (380 уникальных статей)
        │
        ▼
  generate_questions.py: YandexGPT → 5-10 вопросов к каждой статье
        │
        ▼
  Чанкинг (1000 символов/чанк) → 2467 чанков
        │
        ▼
  Эмбеддинги (Yandex, 9 запросов/сек, ~4 мин)
        │
        ▼

```

## 6. Логика классификации

Два уровня ключевых слов (бесплатно, без LLM):

| Уровень          | Что определяет                                    | Перебивает сессию? |
|------------------|---------------------------------------------------|--------------------|
| `_COUNTRY_NAMES` | «казахстан», «минск», «турция» — явное упоминание | ✅ Да               |
| `_REGION_TERMS`  | «эдо», «тенге», «мрп» — контекст                  | ❌ Нет              |

12 регионов: ru, kz, by, uz, am, md, ge, lt, rs, kg, tj, tr.

---

## 7. Гибридный поиск

```
Векторный поиск (Yandex embeddings)
  → cosine similarity к 2467 чанкам
  → региональный фильтр (RU = универсальный, видят все)
  → текстовый буст: +0.15 за слово запроса в заголовке
     (для не-RU: ×0.3 штраф для RU-статей)
  → top-3 результата
  → fallback: РФ если не найдено
```

---

## 8. Региональная логика

- При /start — выбор региона из 12 стран
- Регион сессии сохраняется на все последующие вопросы
- Явное упоминание страны в вопросе перебивает сессию
- Статьи с `region=None` (универсальные) видны всем
- Статьи с `region="kz"` видны только KZ-пользователям
- RU-статьи — универсальный контент, доступный всем регионам
- Для регионов без своей БЗ — fallback на РФ с пометкой

---

## 9. Ключевые решения

1. **380 реальных статей** — контент с pro.yandex, не ручной пересказ
2. **Генерация вопросов** — YandexGPT создал 3762 пользовательских вопроса к статьям для улучшения поиска
3. **Гибридный поиск** — векторный + текстовый буст по заголовкам (решил проблему слабых Yandex embeddings)
4. **12 регионов** — полная карта присутствия Яндекс Про
5. **Rate limiter** — token bucket 9 req/s, ноль 429 ошибок при индексации
6. **Хеш-автоиндексация** — entrypoint.sh сам определяет когда нужно переиндексировать
7. **Экранирование Markdown** — безопасная отправка LLM-ответов в Telegram
